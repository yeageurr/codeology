HTML Introduction



HTML is the standard markup language for creating Web Pages.



What is HTML?

  • HTML stands for Hyper Text Markup Language.
  • HTML is the standard markup language for creating Web pages
  • HTML describes the structure of a Web page
  • HTML consists of a series of elements
  • HTML elements tell the browser how to display the content
  • HTML elements label pieces of content such as "this is a heading", "this is a paragraph", "this is a link", etc.

HTML Tags and Elements



HTML documents are made up of elements, which consist of tags, attributes, and content.



HTML Elements and Tags

HTML tags are the fundamental elements of HTML used for defining the structure of the document. These are letters or words enclosed by angle brackets (< and >).

Usually, most of the HTML tags contain an opening and a closing tag. Each tag has a different meaning, and the browser reads the tags and displays the contents enclosed by them accordingly.

HTML comments are non-executable lines of code that do not display on the webpage and are used to add notes or explanations. It is a good practice to add comments into your HTML code, especially in complex documents, to indicate sections of a document and any other notes to anyone looking at the code.

JavaScript Data Types



Data types in JavaScript refers to the types of the values that we are storing or working with. One of the most fundamental characteristics of a programming language is the set of data types it supports.

In JavaScript, there are different types of data that can be stored and manipulated in variables. These data types are categorized into primitive and non-primitive (reference) types.



1. Primitive Data Types

Primitive data types are basic, immutable, and stored directly in memory. When a variable holds a primitive value, it contains the actual data, not a reference to it. If you assign a primitive variable to another variable, a copy of the value is created, meaning modifying one does not affect the other.

1.1 Strings

  • Strings represent sequences of characters and are enclosed in single ('), double ("), or backticks (``) for template literals.
  • Strings are immutable, meaning you cannot change a character directly. Any modifications create a new string.

Example:

let greeting = "Hello World!";

let squad = "Yeagerists";

let message = `Shinzou wo sasageyo, ${squad}!`; // Template literals allow interpolations

Immutable Behavior Example:

let mystring = "Banana";

mystring[0] = "A"; // This won't work (strings are immutable)

console.log(mystring); // Output: "Hello"

1.2 Numbers (numeric data)

  • JavaScript has only one Number type for both integers and floating-point numbers.
  • It includes special values like NaN (Not a Number), Infinity, and -Infinity.

Example:

let age = 20; // integer

let grade = 1.5 // floating-point

let result = NaN; // Not a number

let large = Infinity; // Represents an infinitely large number

Operations with Numbers:

console.log(5 + 3); // Output: 8

console.log(10 / 0); // Output: Infinity

1.3 Boolean (True/False)

  • Represents true or false values, used for logic and decision-making.
  • Often used in conditional statements.

Example:

let isRaining = false;

let isEmpty = true;


if(isEmpty){
    console.log('Refill.');

    console.log('Drink Coffee.');

1.4 Undefined (Unassigned Variable)

  • If a variable is declared but not assigned a value, it is automatically undefined.
  • Undefined is different from null. Undefined means "no value assigned," while null means "intentional empty value."

Example:

let name;

console.log(name); // Output: undefined

1.5 Null (Intentional Empty Value)

  • Represents the intentional absence of a value.
  • Often used to reset a variable or indicate that an object should be empty.

Example:

let name = null;

console.log(name); // Output: null

1.6 BigInt (Large Numbers)

  • Introduced in ES11 (ES2020) to store very large numbers beyond Number.MAX_SAFE_INTEGER.
  • A BigInt is created by appending "n" to the number.

1.7 Symbol (Unique Identifiers)

  • Used to create unique values that wonâ€™t collide with others.

Example:

1. Non-Primitive (Reference) Data Types

Non-primitive data types are mutable and stored by reference. This means that when you assign a non-primitive variable to another variable, both variables point to the same memory location, not a copy.

2.1 Objects (Key-Value Pairs)

  • Objects store multiple values in key-value pairs.
  • Values can be any data type, including functions.

  • Arrays store multiple values in an indexed format.
  • Can contain different data types.

Example:

console.log(ranks[2]); // Output: Lieutenant General


2.3 Functions (Reusable Block of Codes)

  • Functions are objects in JavaScript.
  • They can be assigned to variables, passed as arguments, and returned.

Example:

    return `Hello, Captain ${name}!`;

2.4 Date (Represents Date and Time)

  • JavaScript provides the Date object to handle date and time.

Example:

console.log(today); // Output: current date and time



Conclusion

Understanding primitive and non-primitive data types is essential for writing efficient JavaScript code.

  • Primitive types store values directly and are immutable.
  • Non-primitive types store references to objects in memory and are mutable.
  • Always be aware of the difference between storing by value vs. reference to avoid unintended side effects.
  • Examples of Primitive Types are: String, Numbers, Booleans, Null, Undefined, Symbol, and BigInt.
  • Examples of Non-Primitive Types are: Objects, Arrays, Functions, and Date.


JavaScript Variables



JavaScript variables are used to store data that can be changed later on. These variables can be thought of as named containers. You can place data into these containers and then refer to the data simply by naming the container.

JavaScript provides different ways to declare variables, and understanding how they work is crucial for writing efficient and bug-free programs.



Declaring Variables

  • var â€” The old way (function scope, can be redeclared)
  • let â€” The modern way (block-scoped, cannot be redeclared in the same scope)
  • const â€” Used for constants (block-scoped, cannot be reassigned)

Syntax:

let name = 'Zeke Yeager'; // Using let
var age = 29; // Using var
const Pi = 3.1416 // Using const

Key Differences Between let, var, and const variables

  • Scope: var is function-scoped, while let and const are block-scoped.
  • Redeclaration: var can be redeclared within the same scope, but let and const cannot.
  • Reassignment: var and let can be reassigned, whereas const cannot be changed after initialization.

Example:

function myfunction() {
  if (true) {
    var x = 10; // Function-scoped
    let y = 20; // Block-scoped
    const z = 30; // Block-scoped
  };

  console.log(x); // Output: 10 (works because x is function-scoped)
  console.log(y); // Output: undefined (error because y is block-scoped)
  console.log(z); // Output: undefined (error because z is block-scoped)
};

myfunction();


Variable Hoisting

Hoisting is JavaScriptâ€™s default behavior of moving variable declarations to the top of their scope before execution.
This means you can use functions and variables before they are declared in the code.

Hoisting with var:

Variables declared with var are hoisted but not initialized. They get a default value of undefined.

console.log(a); // Output: undefined (Hoisted, but not assigned)
var a = 10;
console.log(a); // Output: 10

Even though a is used before its declaration, JavaScript does not throw an error because var a; is hoisted to the top.


Hoisting with let and const

Variables declared with let and const are also hoisted but are not initialized.
Accessing them before their declaration results in a ReferenceError.

console.log(a); // Output: ReferenceError: Cannot access 'a' before initialization
let a = 10;

Hoisting with functions

Function declarations are fully hoisted, meaning they can be called before their definition.

sayHi(); // Output: "Hi miss beautiful! :>"

function sayHi() {
  console.log('Hi miss beautiful! :>');
};

However, function expressions (assigned to variables) are hoisted but not initialized, just like let and const.

sayHi(); // Output: TypeError: sayHi is not a function

var sayHi = function() {
  console.log('Hi miss beautiful! :>');
};


Variable Naming Rules

  • Must start with a letter, _ (underscore), or $ (dollar sign).
  • Cannot start with a number.
  • JavaScript is case-sensitive (name and Name are different).
  • Reserved keywords (e.g., let, const, var, if) cannot be used.

Examples of Valid and Invalid Variable Names

let $dollarValue = 57.3953; // This name is also valid
let userName = "_rivaii"; // This name is also valid


let 1stUser = "_rivaii"; // Invalid, cannot start with a number
let var = "_rivaii"; // Invalid, reserved keyword

Checking the Data Type of a variable

In Javascript, we use the typeof built-in function when we want to check what data type a variable holds.


let age = 20;
console.log(typeof age); // Output: number

let name = 'Armin Arlert';
console.log(typeof name); // Output: string

console.log(typeof isTitan); // Output: boolean

Variable Scope

Scope determines where a variable can be accessed. JavaScript has:

  • Global Scope â€” Variables declared outside functions.
  • Function Scope â€” Variables declared inside a function.
  • Block Scope â€” Variables declared inside {} using let or const.

Examples of Global, Function, and Block Scope

let globalVariable = 'Global'; // Global Variable

function myFunction() {
  var function_variable = "I'm inside a function"; // A function-scoped variable
  console.log(globalVariable); // Output: 'Global' (global variables are accessible inside and outside a function)
};

if (true) {
  let block_variable = "I'm inside a block"; // A block-scoped variable
  console.log(block_variable); // Output: "I'm inside a block" (accessible since it is inside the if-block)
};


console.log(globalVariable); // Output: 'Global' (global variable, accessible anywhere)
console.log(function_variable); // Output: ReferenceError: function_variable is not defined (function-scoped variable, not accessible outside the function where it is declared)

console.log(block_variable); // Output: ReferenceError: block_variable is not defined (block-scoped variable, not accessible outside the if-block where it is declared)


Conclusion

  • Use let for variables that change, const for constants, and avoid var in modern JavaScript.
  • Remember hoisting behavior and avoid accessing variables before declaration.
  • Choose meaningful variable names for better readability.
  • Understand scope rules to avoid unexpected behavior in your programs.


JavaScript Operators



JavaScript operators are used to perform operations on variables and values. They can be categorized based on their functionality.
Let us take a simple expression 4 + 5 is equal to 9. Here 4 and 5 are called operands, and + is called the operator.



1. Arithmetic Operators

Arithmetic operators are used to perform mathematical calculations like addition, subtraction, multiplication, division, and more. These operators take numerical values (operands) and return a result based on the operation performed.

+(Addition) â€” Adds two numbers or concatenates strings.

let sum = 10 + 5; // sum is 15
let name = "Porco" + " " + "Galliard"; // name is "Porco Galliard"

-(Subtraction) â€” Subtracts the second number from the first.

let diff = 10 - 5; // diff is 5

*(Multiplication) â€” Multiplies two numbers.

let product = 10 * 5; // product is 50

/(Division) â€” Divides the first number by the second.

let quotient = 10 / 5; // quotient is 2

%(Modulus) â€” Returns the remainder when one number is divided by another.

let remainder = 10 % 5; // remainder is 1

**(Exponentiation) â€” Raise a number to a power.

let power = 3 ** 5; // power is 243 (3 raise to the power of 5)


2. Assignment Operators

Assignment operators are used to assign values to variables and modify them in place.

=(Assignment) â€” Assigns a value to a variable.

let age = 75; // age is assigned the value 10

+=(Addition Assignment) â€” Adds a value to the variable and assigns the new value.

age += 5; // age is now 80 (same as age = age + 5)

-=(Subtraction Assignment) â€” Subtracts a value from the variable.

age -= 60; // age is now 20 (same as age = age - 60)

*=(Multiplication Assignment) â€” Multiplies the variable by a value.

age *= 2; // age is now 40 (same as age = age * 2)

/=(Division Assignment) â€” Divides the variable by a value.

age /= 2; // age is now 20 (same as age = age / 2)

%=(Modulus Assignment) â€” Assigns the remainder of division.

age %= 6; // age is now 2 (same as age = age % 6)


3. Comparison Operators

Comparison operators compare two values and return a boolean (true or false). They are commonly used in conditional statements.

==(Equal to) â€” Compares values but ignores data types.

console.log(10 == "10"); // Output: true (compares values only)

===(Strict equal to) â€” Compares both value and type.

console.log(10 == "10"); // Output: false (compares value and data type)

!=(Not equal to) â€” Checks if two values are different.

console.log(10 != 5); // Output: true (10 is not equal to 5)

!==(Strict not equal to) â€” Checks if value and type are both different.

console.log(10 !== 5); // Output: true (value is the same, but type is different)

>(Greater than) â€” Checks if the left value is greater.

console.log(10 > 5); // Output: true

<(Less than) â€” Checks if the left value is smaller.

console.log(10 < 5); // Output: false

>=(Greater than or equal to) â€” Checks if the left value is greater or equal.

console.log(10 >= 10); // Output: true

<=(Less than or equal to) â€” Checks if the left value is smaller or equal.

console.log(10 <= 5); // Output: false



4. Logical Operators

Logical operators allow combining multiple conditions, often used in if statements.

&&(Logical AND) â€” Returns true if both conditions are true.

console.log(true && false); // Output: true
console.log(15 > 10 && 5 < 3); // Output: false

||(Logical OR) â€” Returns true if at least one condition is true.

console.log(true || false); // Output: true
console.log(15 > 10 || 5 < 3); // Output: true

!(Logical NOT) â€” Reverses the boolean value.

console.log(!true); // Output: false
console.log(!(15 > 10)); // Output: false


5. Bitwise Operators

JavaScript If...Else



The if...else statement in JavaScript is used to execute a block of code based on a condition. If the condition is true, the if block runs; otherwise, the else block runs.

Basic If Statement

The if statement executes a block of code only if the condition is true.

let age = 18;

if (age >= 18) {
  console.log("You are an adult.");
};

Output: "You are an adult."


If...Else Statement

The else block runs if the if condition is false.

let age = 15;

if (age >= 18) {
  console.log("You are an adult.");
} else {
  console.log("You are a minor.");
};

Output: "You are a minor."


If...Else If...Else Statement

The else block runs if the if condition is false.

let grade = 85;

if (grade >= 90) {
  console.log("Grade: A");
} else if (grade >= 80){
  console.log("Grade: B");
} else if (grade >= 70){
  console.log("Grade: C");
} else {
  console.log("Grade: F");
};

Output: "Grade: B"


Nested If Statement

An if statement inside another if statement.

let num = 5;

if (num > 0) {
  console.log("Positive number");
  if (num % 2 === 0){
    console.log("Even Number");
  };
};

Output:
"Positive Number"
"Even Number"
