How JavaScript was developed?

In 1995, a young programmer named Brendan Eich was given just 10 days to create a lightweight scripting language for Netscape Navigator, one of the earliest web browsers. His goal was simple: make web pages more interactive. The result? A language initially called Mocha, then LiveScript, and finally JavaScript, after a marketing deal with Sun Microsystems to ride on Java’s popularity.

By 1997, JavaScript was gaining traction, but there was a problem—different browsers started implementing their own versions, causing inconsistencies. To solve this, it was submitted to ECMA International, leading to the first official standard: ECMAScript (ES1). As the web grew, so did JavaScript. In 1999, ECMAScript 3 (ES3) introduced powerful new features, making JavaScript more useful for developers.

Around the same time, Microsoft’s Internet Explorer implemented its own version, further fueling JavaScript’s rise. Fast forward to 2005, and a game-changing concept called AJAX (Asynchronous JavaScript and XML) was introduced. For the first time, websites could load data dynamically without refreshing the page, leading to the creation of web applications like Google Maps and Gmail. JavaScript was no longer just a simple scripting language—it was shaping the future of the web.


By 2009, JavaScript was facing new challenges. Developers wanted cleaner, more efficient code. The release of ECMAScript 5 (ES5) introduced improvements like JSON support, new array methods, and strict mode, making JavaScript easier to use and less error-prone.

Then came 2015, a defining moment in JavaScript’s history. ECMAScript 6 (ES6), also known as ES2015, introduced features like let and const, arrow functions, and classes, making JavaScript more modern and powerful. It was a major shift, allowing developers to write cleaner, more structured code.

Since then, JavaScript has continued to evolve every year, bringing in async/await for better asynchronous programming, optional chaining for cleaner code, and performance improvements for modern web applications. Today, it powers everything from simple websites to advanced applications like Netflix, Facebook, and even AI-driven platforms.

JavaScript Data Types



Data types in JavaScript refers to the types of the values that we are storing or working with. One of the most fundamental characteristics of a programming language is the set of data types it supports.

In JavaScript, there are different types of data that can be stored and manipulated in variables. These data types are categorized into primitive and non-primitive (reference) types.



1. Primitive Data Types

Primitive data types are basic, immutable, and stored directly in memory. When a variable holds a primitive value, it contains the actual data, not a reference to it. If you assign a primitive variable to another variable, a copy of the value is created, meaning modifying one does not affect the other.

1.1 Strings

  • Strings represent sequences of characters and are enclosed in single ('), double ("), or backticks (``) for template literals.
  • Strings are immutable, meaning you cannot change a character directly. Any modifications create a new string.

let message = `Shinzou wo sasageyo, ${squad}!`; // Template literals allow interpolations

Immutable Behavior Example:

console.log(mystring); // Output: "Hello"

1.2 Numbers (numeric data)

  • JavaScript has only one Number type for both integers and floating-point numbers.
  • It includes special values like NaN (Not a Number), Infinity, and -Infinity.

Example:

let age = 20; // integer

Operations with Numbers:

JavaScript Variables



JavaScript variables are used to store data that can be changed later on. These variables can be thought of as named containers. You can place data into these containers and then refer to the data simply by naming the container.

JavaScript provides different ways to declare variables, and understanding how they work is crucial for writing efficient and bug-free programs.



Declaring Variables

  • var — The old way (function scope, can be redeclared)
  • let — The modern way (block-scoped, cannot be redeclared in the same scope)
  • const — Used for constants (block-scoped, cannot be reassigned)

Syntax:

let name = 'Zeke Yeager'; // Using let
var age = 29; // Using var
const Pi = 3.1416 // Using const

Key Differences Between let, var, and const variables

  • Scope: var is function-scoped, while let and const are block-scoped.
  • Redeclaration: var can be redeclared within the same scope, but let and const cannot.
  • Reassignment: var and let can be reassigned, whereas const cannot be changed after initialization.

Example:

function myfunction() {
  if (true) {
    var x = 10; // Function-scoped
    let y = 20; // Block-scoped
    const z = 30; // Block-scoped
  };

  console.log(x); // Output: 10 (works because x is function-scoped)
  console.log(y); // Output: undefined (error because y is block-scoped)
  console.log(z); // Output: undefined (error because z is block-scoped)
};

myfunction();


Variable Hoisting

Hoisting is JavaScript’s default behavior of moving variable declarations to the top of their scope before execution.
This means you can use functions and variables before they are declared in the code.

Hoisting with var:

Variables declared with var are hoisted but not initialized. They get a default value of undefined.

console.log(a); // Output: undefined (Hoisted, but not assigned)
var a = 10;
console.log(a); // Output: 10

Even though a is used before its declaration, JavaScript does not throw an error because var a; is hoisted to the top.


Hoisting with let and const

Variables declared with let and const are also hoisted but are not initialized.
Accessing them before their declaration results in a ReferenceError.

console.log(a); // Output: ReferenceError: Cannot access 'a' before initialization
let a = 10;

Hoisting with functions

Function declarations are fully hoisted, meaning they can be called before their definition.

sayHi(); // Output: "Hi miss beautiful! :>"

function sayHi() {
  console.log('Hi miss beautiful! :>');
};

However, function expressions (assigned to variables) are hoisted but not initialized, just like let and const.

sayHi(); // Output: TypeError: sayHi is not a function

var sayHi = function() {
  console.log('Hi miss beautiful! :>');
};


Variable Naming Rules

  • Must start with a letter, _ (underscore), or $ (dollar sign).
  • Cannot start with a number.
  • JavaScript is case-sensitive (name and Name are different).
  • Reserved keywords (e.g., let, const, var, if) cannot be used.

Examples of Valid and Invalid Variable Names

let $dollarValue = 57.3953; // This name is also valid
let userName = "_rivaii"; // This name is also valid

In Javascript, we use the typeof built-in function when we want to check what data type a variable holds.

console.log(typeof age); // Output: number

let name = 'Armin Arlert';

Variable Scope

Scope determines where a variable can be accessed. JavaScript has:

  • Global Scope — Variables declared outside functions.
  • Function Scope — Variables declared inside a function.
  • Block Scope — Variables declared inside {} using let or const.

Examples of Global, Function, and Block Scope

let globalVariable = 'Global'; // Global Variable

function myFunction() {
  var function_variable = "I'm inside a function"; // A function-scoped variable
  console.log(globalVariable); // Output: 'Global' (global variables are accessible inside and outside a function)
};

if (true) {
  let block_variable = "I'm inside a block"; // A block-scoped variable
  console.log(block_variable); // Output: "I'm inside a block" (accessible since it is inside the if-block)
};


console.log(globalVariable); // Output: 'Global' (global variable, accessible anywhere)
console.log(function_variable); // Output: ReferenceError: function_variable is not defined (function-scoped variable, not accessible outside the function where it is declared)

console.log(block_variable); // Output: ReferenceError: block_variable is not defined (block-scoped variable, not accessible outside the if-block where it is declared)


Conclusion

  • Use let for variables that change, const for constants, and avoid var in modern JavaScript.
  • Remember hoisting behavior and avoid accessing variables before declaration.
  • Choose meaningful variable names for better readability.
  • Understand scope rules to avoid unexpected behavior in your programs.


JavaScript Operators



JavaScript operators are used to perform operations on variables and values. They can be categorized based on their functionality.
Let us take a simple expression 4 + 5 is equal to 9. Here 4 and 5 are called operands, and + is called the operator.



1. Arithmetic Operators

Arithmetic operators are used to perform mathematical calculations like addition, subtraction, multiplication, division, and more. These operators take numerical values (operands) and return a result based on the operation performed.

+(Addition) — Adds two numbers or concatenates strings.

let sum = 10 + 5; // sum is 15
let name = "Porco" + " " + "Galliard"; // name is "Porco Galliard"

-(Subtraction) — Subtracts the second number from the first.

let diff = 10 - 5; // diff is 5

*(Multiplication) — Multiplies two numbers.

let product = 10 * 5; // product is 50

/(Division) — Divides the first number by the second.

let quotient = 10 / 5; // quotient is 2

%(Modulus) — Returns the remainder when one number is divided by another.

let remainder = 10 % 5; // remainder is 1

**(Exponentiation) — Raise a number to a power.

let power = 3 ** 5; // power is 243 (3 raise to the power of 5)


2. Assignment Operators

Assignment operators are used to assign values to variables and modify them in place.

=(Assignment) — Assigns a value to a variable.

let age = 75; // age is assigned the value 10

+=(Addition Assignment) — Adds a value to the variable and assigns the new value.

age += 5; // age is now 80 (same as age = age + 5)

-=(Subtraction Assignment) — Subtracts a value from the variable.

age -= 60; // age is now 20 (same as age = age - 60)

*=(Multiplication Assignment) — Multiplies the variable by a value.

age *= 2; // age is now 40 (same as age = age * 2)

/=(Division Assignment) — Divides the variable by a value.

age /= 2; // age is now 20 (same as age = age / 2)

%=(Modulus Assignment) — Assigns the remainder of division.

age %= 6; // age is now 2 (same as age = age % 6)


3. Comparison Operators

Comparison operators compare two values and return a boolean (true or false). They are commonly used in conditional statements.

==(Equal to) — Compares values but ignores data types.

console.log(10 == "10"); // Output: true (compares values only)

===(Strict equal to) — Compares both value and type.

console.log(10 == "10"); // Output: false (compares value and data type)

!=(Not equal to) — Checks if two values are different.

console.log(10 != 5); // Output: true (10 is not equal to 5)

!==(Strict not equal to) — Checks if value and type are both different.

console.log(10 !== 5); // Output: true (value is the same, but type is different)

>(Greater than) — Checks if the left value is greater.

console.log(10 > 5); // Output: true

<(Less than) — Checks if the left value is smaller.

console.log(10 < 5); // Output: false

The Story of PHP: How It Became a Web Giant

Back in 1994, a Danish-Canadian programmer named Rasmus Lerdorf had a simple problem—he wanted to track visits to his personal website. To solve this, he wrote a small set of scripts in C and called them "Personal Home Page Tools" (PHP Tools). Little did he know, this small project would grow into one of the most powerful programming languages for the web.

As Rasmus continued improving PHP, more developers became interested. By 1995, he released PHP to the public, allowing others to use and modify it. At this stage, PHP was just a collection of tools for handling simple web tasks, but its potential was undeniable.

By 1997, PHP had evolved into PHP/FI (Forms Interpreter), adding basic support for databases. Developers worldwide began adopting it, seeing its ability to create dynamic web pages. However, it still lacked a proper structure—until two Israeli developers, Zeev Suraski and Andi Gutmans, stepped in.

Seeing PHP’s potential, Suraski and Gutmans completely rewrote the core of PHP, leading to the release of PHP 3 in 1998. This version transformed PHP into a real programming language, supporting object-oriented programming and powerful extensions. But the real game-changer came in 2000 with PHP 4. It introduced the Zend Engine, optimizing performance and making PHP more efficient for large-scale applications. This was the version that made PHP a serious contender in web development.

As the internet grew, so did PHP. In 2004, PHP 5 was released, bringing advanced object-oriented programming (OOP), better security, and improved database support. It became the backbone of WordPress, Facebook, and Wikipedia, powering millions of websites worldwide. Fast forward to 2015, and PHP underwent another major transformation with PHP 7. Thanks to the Zend Engine 3, PHP became almost twice as fast as its previous versions, making it even more attractive for developers.

Then, in 2020, PHP 8 was released, introducing Just-In-Time (JIT) compilation, further boosting performance and efficiency. PHP had come a long way from a simple set of tools to a powerful and scalable language driving the modern web.

  • int age = 20; // Declaring and initializing an integer variable
    float height = 6.2; // Float variable
    char grade = 'A'; // Char variable

  • You can also declare multiple variables of the same type in a single line:

    int x = 5, int y = 10, int z = 15;// Multiple variables

  • Or declare first, then assign a value later:

    int score; // Declaration
    score = 100 // Assignment


Types of Variables

  1. Local Variables – Declared inside functions or blocks, accessible only within that block.

    void function() {
      int localVar = 10; // Only accessible inside this function
    }


  2. Global Variables – Declared outside all functions, accessible by all functions in the program.

    int = globalVar = 25; // Global variable


    void display() {
      printf("%d", globalVar); // Accessible here
    }


  3. Static Variables – Retain their value between function calls.

    void counter() {
      static int count = 0;
      count++;
      printf("%d", count); // Will keep incrementing across calls
    }


  4. Constants – Variables declared using const cannot be modified after being assigned.

    const float PI = 3.14; // Constant value



Variables are essential in C, acting as containers for data that the program manipulates. Choosing the right type and scope ensures your code is efficient and organized.

C if...else Statements



The if...else statements in C are used to perform conditional execution of code blocks. They allow programs to make decisions based on whether a given condition is true or false.



1. The Basic if Statement

Use the if statement when you want to execute a code block only if a condition is true.

int age = 20;


if (age >= 18) {
printf("You are eligible to vote.");
}

The message is only printed if the condition (age >= 18) is true.



2. The if..else Statement

The if..else statement is used when there are two possible outcomes — one if the condition is true, and another if it's false.

int number = 10;


if (number % 2 == 0) {
  printf("Even number.");
} else {
  printf("Even number.");
}

The program chooses between printing "Even number" or "Odd number" depending on the condition.



3. The if...else if...else Statement

This is used when you have multiple conditions to check. It evaluates each condition from top to bottom and executes the first one that is true.

int score = 85;


if (score >= 90) {
  printf("Grade: A");
} else if (score >= 80) {
  printf("Grade: B");
} else {
  printf("Grade: C");
}

The program checks each condition in order and executes the matching block. Only one block runs, even if multiple conditions are true.



4. The nested if Statement

A nested if is an if statement inside another if or else block, used for checking multiple layers of conditions.

int age = 20;
int hasLicense = 1;


if (age >= 18) {
  if (hasLicense) {
    printf("You can drive.");
  } else {
    printf("You need a license to drive.");
  }

} else {
  printf("You are too young to drive.");
}

The program checks each condition in order and executes the matching block. Only one block runs, even if multiple conditions are true.



C’s if...else structure is essential for making decisions in programs. It adds logic and control to how a program runs based on different inputs and situations.

Loops



Loops in C are used to repeat a block of code multiple times. Instead of writing the same code again and again, loops help execute it efficiently based on conditions. C supports three main types of loops: for, while, and do...while.



1. The for Loop

The for loop is best when you know exactly how many times you want to repeat a block of code.

Syntax:

for (initialization; condition; update) {
  // Code to be executed
}

Example:

for (int i = 1; i <= 5; i++) {
  printf("Count: %d\n", i);
}

Output:

1
2
3
4
5

The loop starts with i = 1 and keeps running while i is less than or equal to 5. Each time the loop runs, i increases by 1, so the output becomes 1 through 5.



2. The while Loop

Use a while loop when you want to repeat a block of code as long as a condition is true, and the number of iterations is not known beforehand.

Syntax:

while (condition) {
  // Code to be executed
}

Example:

int i = 1;

while (i <= 5) {
  printf("Count: %d\n", i);
}

Output:

1
2
3
4
5

The loop begins with i = 1 and continues to run as long as i <= 5. Since i increases by 1 after each iteration, the output displays the numbers from 1 through 5.



3. The do...while Loop

This loop is similar to while, but it executes the code block at least once, even if the condition is false at first.

Syntax:

do {
  // Code to be executed
} while (condition);

Example:

int i = 1;

do {
  printf("Count: %d\n", i);
  i++;
} while (i <= 5);

Output:

1
2
3
4
5

This loop also prints 1 through 5 because it starts with i = 1 and increments i by 1 each time. The key difference is that the do block runs at least once, so even if the condition were false at the start, the first iteration would still execute. In this case, the condition remains true until i > 5, so it prints 1 to 5.



4. Loop Control Statements

Loop control statements are special keywords in C that let you control the flow of a loop. They allow you to either skip an iteration (continue) or exit the loop early (break), giving you more flexibility in how your loops behave.

break: Exits the loop immediately.

for (int i = 1; i <= 5; i++) {
  if (i == 3) {
    break;
  }

  printf("%d\n", i);
}
// Output: 1, 2

continue: Skips the current iteration and moves to the next.

for (int i = 1; i <= 5; i++) {
  if (i == 3) {
    continue;
  }

  printf("%d\n", i);
}
// Output: 1, 2, 4, 5



Loops in C provide a powerful way to perform repetitive tasks efficiently. Whether you're iterating a known number of times or looping based on dynamic conditions, using the right loop structure can make your code cleaner and faster.
